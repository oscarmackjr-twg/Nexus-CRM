<objective>
Close the deal-child side-door (D-19): `DealCounterpartyService` and `DealFundingService` currently scope only by `org_id`, so a user from another group can read and mutate counterparties and funding entries belonging to a deal they cannot see (IDOR — Bugs 7, 8). Replace their `_get_deal_or_404` usage with the shared `require_deal_readable` (reads) / `require_deal_writable` (writes) guards so child visibility inherits the parent deal's access rules. Then apply the minor frontend change so the deal detail page distinguishes a 403 ("no permission") from a 404 ("not found").

Purpose: Lock ACCESS-02 / ACCESS-07 for deal child entities (no record-existence leakage via a side door) and deliver the success-criterion-#6 UX distinction on the client.
Output: Guarded `counterparties.py` + `funding.py`, child-entity cross-team tests, and the `DealDetailPage.jsx` 403/404 patch (human-verified).
</objective>
