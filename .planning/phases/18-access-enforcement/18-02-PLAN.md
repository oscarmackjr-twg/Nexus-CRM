## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated client → deal detail/mutation endpoints | JWT-bearing caller requests a specific deal id; server must distinguish exists-but-forbidden (403) from absent (404) and gate writes/deletes by role |
| update payload → owner_id / team assignment | Caller-supplied `owner_id` could move a record out of their access if not gated to admin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-05 | Information Disclosure | `get_deal` / `_get_deal_or_404` 404-vs-403 | mitigate | Load-then-decide: existence check scoped to org_id only, then `can_read_deal` decides 403; in-org out-of-scope ids return 403 not 404 (D-01/D-02, ACCESS-07) |
| T-18-06 | Elevation of Privilege | supervisor editing other group's deal via `is_manager_plus` | mitigate | Replace with `can_write_deal` which requires `deal.team_id == user.team_id` for supervisors (Bug 4/6 closed) |
| T-18-07 | Elevation of Privilege | non-owner delete (supervisor deletes member's deal) | mitigate | `can_delete_deal` returns owner-or-admin only; supervisor delete of others' deals -> 403 (D-08, criterion #3) |
| T-18-08 | Tampering | `owner_id` reassignment to escalate/relocate a record | mitigate | Early guard `data.owner_id is not None and not is_admin(user) -> 403` (D-13) |
| T-18-09 | Tampering | `team_id` reassignment moving record out of scope | accept | DealUpdate schema has no `team_id` field — structurally impossible to set via payload (verified) |

No package-manager installs occur in this plan (no `T-18-SC` row required).
