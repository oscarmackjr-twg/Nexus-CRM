## Spacing Scale

Declared values (must be multiples of 4). Source: Tailwind defaults; confirmed consistent with existing component usage in ContactDetailPage and DealDetailPage.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, chip internal padding |
| sm | 8px | Input internal padding, compact row gaps |
| md | 16px | Card body padding, form field gaps, section spacing within a card |
| lg | 24px | Card header padding, tab panel top padding |
| xl | 32px | Gap between Profile section cards |
| 2xl | 48px | Page top/bottom padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Milestone date grid rows: 12px vertical padding per row (between `sm` and `md`) to keep the 9-row grid compact without crowding
- Fund quick-create modal internal padding: 24px (lg) on all sides
- Touch targets (Edit/Save buttons): minimum 36px height

## Typography

Source: `frontend/src/styles.css` (IBM Plex Sans), existing component patterns in ContactDetailPage. No custom `fontSize` in tailwind.config.js — Tailwind defaults apply.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Form labels, field values, milestone date values, table cells |
| Label | 12px (text-xs) | 500 (medium) | 1.4 | Section sub-headings within cards, currency code badge, chip labels |
| Heading | 16px (text-base) | 600 (semibold) | 1.3 | Card titles (CardTitle), tab trigger labels |
| Display | 20px (text-xl) | 600 (semibold) | 1.2 | Deal name at top of DealDetailPage header |

Two weights declared: 400 (normal) and 600 (semibold). Medium (500) used only for label role micro-text to distinguish it from body without adding a third full weight tier.

## Color

Source: `frontend/src/styles.css` CSS custom properties; `frontend/tailwind.config.js` token mapping.

| Role | Token | HSL Value (light) | Usage |
|------|-------|-------------------|-------|
| Dominant (60%) | `--background` | hsl(220 33% 98%) — near-white blue-grey | Page background, tab panel surface |
| Secondary (30%) | `--card` | hsl(0 0% 100%) — white | Section cards, Fund quick-create modal background, dropdown panels |
| Accent (10%) | `--primary` | hsl(239 84% 67%) — indigo | See reserved list below |
| Destructive | `--danger` | hsl(0 84% 60%) — red | Destructive actions only (see below) |

Accent (`--primary` indigo) reserved for:
1. Active tab trigger underline / selected state
2. Edit button text color (ghost variant)
3. Save button background (primary variant)
4. `+ New fund` inline trigger text/icon
5. Chip remove icon (×) hover state
6. Focus ring on all interactive inputs (`--ring` = same indigo)

Muted (`--muted` hsl(220 20% 96%)) used for: read-only field backgrounds, milestone row alternating shade, disabled states.

Muted-foreground (`--muted-foreground` hsl(220 9% 46%)) used for: null/empty field display (`—`), placeholder text, secondary descriptor text.

Destructive (`--danger`) reserved for:
- No destructive actions exist in Phase 4 (Fund entity has no delete; deals are not deleted from DealDetailPage). The danger token is declared for completeness but no UI element uses it this phase.

## Layout Contract

### DealDetailPage — Tab Structure

```
DealDetailPage
├── Header row — deal name (Display), pipeline stage badge, company link
└── Tabs (full-width)
    ├── [0] Profile     — 5 section cards (this phase)
    ├── [1] Activity    — existing activity feed (moved from 3-col grid)
    ├── [2] AI Insights — existing DealScoreCard (moved from 3-col grid)
    └── [3] Tasks       — existing tasks list (moved from 3-col grid)
```

Tab label rules:
- Full labels at all viewport widths: "Profile", "Activity", "AI Insights", "Tasks"
- If viewport < 480px, abbreviate "AI Insights" to "AI" only
- Active tab: `--primary` bottom border 2px, semibold weight
- Inactive tab: muted-foreground, normal weight

### Profile Tab — Card Order

```
Profile TabsContent
├── [1] Deal Identity card
├── [2] Financials card
├── [3] Process Milestones card
├── [4] Source & Team card
└── [5] Passed / Dead card
```

Cards stack vertically with `gap-xl` (32px) between them. Each card is full width of the tab panel. No multi-column card grid.

## Interaction Contracts

### Per-Card Edit/Save Pattern

Identical to Phase 3 Contact/Company Profile cards:
- Each card has an "Edit" button (ghost, pencil icon, top-right of CardHeader)
- Only one card open in edit mode at a time — opening a second card saves (or cancels) the current one
- Edit mode: form fields render in place; "Save" (primary) and "Cancel" (ghost) buttons appear at card footer
- Save: calls `PATCH /deals/{id}` with only the changed fields (`model_dump(exclude_unset=True)` pattern); on success `toast.success("Deal updated")` + `invalidateQueries(['deal', id])`
- Cancel: resets local state, exits edit mode with no API call
- Saving state: Save button shows spinner, is disabled; Cancel also disabled during save

### Conditional Field Visibility

- `platform_company_id` field: visible in both read and edit mode only when `platform_or_addon === 'addon'`
- In read mode when `platform_or_addon !== 'addon'`: platform_company row is omitted entirely (no empty row)

### Fund Dropdown Auto-Select

After Fund quick-create modal POST succeeds:
1. `invalidateQueries(['funds'])`
2. Set the `fund_id` field value in the Deal Identity card form to the new fund's id
3. Close the modal
No extra user click required.

### Skeleton Loading

While `dealQuery.isLoading`:
- Tab triggers render as Skeleton `h-8 w-24` repeated 4 times
- Each card renders as `Skeleton h-48 w-full` (or per-card height estimate)

While `dealQuery.isError`:
- Show error state in place of Profile tab content (see Copywriting section)
