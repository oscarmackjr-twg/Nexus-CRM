## Anti-Patterns (Executor Must Avoid)

1. **Do not** write `hsl()` wrapper inside CSS variable value for Tailwind-consumed vars. Write `217 60% 25%` not `hsl(217, 60%, 25%)`. Wrong format renders transparent/black in Tailwind.
2. **Do not** edit `tailwind.config.js` тАФ the existing color wiring already handles cascade. Editing breaks the variable indirection.
3. **Do not** use `@import url(...)` for Montserrat in `styles.css`. Use `<link>` in `index.html` per D-01.
4. **Do not** add `@font-face` тАФ no local font files exist. Google Fonts CDN only.
5. **Do not** skip updating the `.dark` block тАФ both `:root` and `.dark` must be updated in the same edit per D-06.
6. **Do not** change `rgba(6, 182, 212, 0.05)` (cyan) in `.surface-grid` тАФ only the first indigo line is being replaced.

## Acceptance Criteria

Phase 7 is done when ALL of the following are true:

1. Every primary button and focus ring in the running app shows `rgb(26, 56, 104)` (navy) instead of indigo тАФ verifiable by browser devtools computed style inspection.
2. Body text renders in Montserrat on all pages тАФ verifiable by devtools computed font-family.
3. CSS variables `--color-brand`, `--color-brand-hover`, `--color-page-bg`, `--color-content-bg`, `--color-text` are defined in `:root` in `styles.css`.
4. `grep -r "indigo\|purple\|violet" frontend/src/` returns zero matches.
5. Existing frontend test suite passes: `cd frontend && npm test -- --run`.
