## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, sort icon inline margin |
| sm | 8px | Inline button padding, badge padding |
| md | 16px | Page header horizontal padding, card padding |
| lg | 24px | Section vertical gap, page top padding |
| xl | 32px | Major layout section breaks |
| 2xl | 48px | Page-level top spacing |
| 3xl | 64px | Not used in this phase |

**Phase-specific exception:** Table row vertical padding is `py-2` (8px) — locked by CONTEXT.md D-05. Horizontal cell padding is `px-4` (16px), on the 4-point scale. No off-scale spacing is used in this phase.

**Touch target minimum:** Row "View" button uses `size="sm"` (`h-8`/ 32px). Grid rows are not touch-primary — this is a desktop deal-team CRM. 32px is acceptable.

## Interaction Contract

### Row Hover & Action Visibility

- `<TableRow>` receives `className="group hover:bg-gray-50 cursor-pointer"`.
- The "View" button cell uses `className="invisible group-hover:visible"` — hidden at rest, visible on row hover.
- Clicking the row navigates to the entity detail page (`/contacts/:id`, `/companies/:id`, `/deals/:id`).
- The "View" button is a secondary affordance — it does NOT stop click propagation; clicking it also navigates.
- No "Edit" button in this phase (CONTEXT.md D-02).

### Column Sort Indicators

- All column headers display a `ChevronsUpDown` icon (Lucide, `h-3 w-3 text-gray-400 ml-1`) at rest — static visual affordance.
- Clicking a sortable column header toggles: unsorted → ascending (`ChevronUp`, accent color) → descending (`ChevronDown`, accent color) → unsorted.
- Sort is client-side only (current page's items), no API params (CONTEXT.md D-01, RESEARCH.md Section 3.6).
- All columns are sortable by default except action column.
- Sort state: `sortKey: string | null`, `sortDir: 'asc' | 'desc'` — lives in the page component (not inside DataGrid), passed as `sortKey / sortDir / onSort` props to avoid state loss on re-render.

### Pagination Bar

- Always visible — even when `pages === 1` (CONTEXT.md D-08, D-09).
- Layout: `flex items-center justify-between` with left side showing `"{total} records · Page {page} of {pages}"` and right side showing per-page selector + Previous + Next buttons.
- Per-page options: 10, 25, 50, 100. Default: 25 (CONTEXT.md D-08).
- Changing per-page resets to page 1.
- Previous button: `disabled` when `page <= 1`.
- Next button: `disabled` when `page >= pages`.
- `keepPreviousData: true` on all list queries — prevents flash-to-skeleton on page change.
- Pagination bar sits below the table with a `border-t border-gray-200` separator.

### Deals Status Filter

- Sits between the page header and the DataGrid.
- Tab options: "All" / "Open" / "Won" / "Lost".
- Layout: `flex gap-1` inline tab bar with `px-3 py-2 text-sm rounded-md` buttons.
- Active tab: `bg-white border border-gray-200 font-semibold text-[#1a3868]` with a `border-b-2 border-[#1a3868]` underline indicator.
- Inactive tab: `text-gray-500 hover:text-gray-700 hover:bg-gray-100`.
- Selecting a tab passes `?status=` to the API. "All" sends no status param (CONTEXT.md D-04).
- Changing filter resets pagination to page 1.

### Loading State

- While `isLoading` is true, render 10 `<Skeleton>` rows of height `h-8` inside the table body.
- Skeleton rows span the full column count.
- Column headers are rendered during loading (not skeletonized).

### Empty State

- When `data.items.length === 0` and `!isLoading`, render a centered empty state inside the table body.
- See copywriting contract below for exact copy.
