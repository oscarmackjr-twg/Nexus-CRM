## Component Contract: RefSelect

The sole new visual component this phase. Wraps the existing `select.jsx` primitive.

### Anatomy

```
[Label text]              ← parent renders label; RefSelect does not render it
[  Select sector  v ]     ← select.jsx primitive (h-10, rounded-xl, border, bg-background)
  [  option 1       ]     ← <option> elements from useRefData data
  [  option 2       ]
  ...
```

### Props Interface (from CONTEXT.md D-06)

```jsx
<RefSelect
  category="sector"          // string — required; maps to GET ?category= param
  value={value}              // string — controlled value
  onChange={onChange}        // function — called with event.target.value
  placeholder="Select sector" // string — first disabled option shown when no value
  disabled={false}           // boolean — optional; propagated to native select
  className=""               // string — optional; passed to cn() override
/>
```

### Loading State

While `useRefData` is fetching (`isLoading === true`): render the `<select>` element with `disabled` prop and a single `<option value="">Loading...</option>`. Use `opacity-50` on the wrapper — no spinner, no skeleton, no layout shift.

### Error State

If `useRefData` returns an error: render the `<select>` with `disabled` prop and a single `<option value="">Failed to load</option>`. Border color changes to `border-danger` (red). No toast, no error boundary — the form field alone signals the error.

### Empty State (no items returned)

Render the `<select>` with a single `<option value="" disabled>No options available</option>`. The field remains interactable but has nothing to select. This can occur when all items in a category are deactivated.

### Option Rendering

Each active `ref_data` item maps to:
```jsx
<option key={item.id} value={item.id}>{item.label}</option>
```

Value is the ref_data UUID (not the `value` slug) — downstream entity fields store the FK to `id`, not to `value`.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| RefSelect loading option | `Loading...` | Shown as single disabled option during fetch |
| RefSelect error option | `Failed to load` | Shown as single disabled option on query error |
| RefSelect empty option | `No options available` | Shown when category returns zero active items |
| RefSelect placeholder | `Select {category display name}` | Passed as prop by caller — e.g., `Select sector`, `Select tier` |
| Primary CTA | Not applicable | Phase 2 adds no form submission buttons |
| Empty state heading | Not applicable | Phase 2 adds no list or page views |
| Error state | `Failed to load` (inline in select) | No toast, no modal |
| Destructive confirmation | Not applicable | Phase 2 has no destructive UI actions |

**Copywriting rules:**
- Use sentence case throughout (not ALL CAPS, not Title Case for option labels).
- Ref data labels render exactly as seeded (e.g., `Financial Services`, `Software & SaaS`) — no truncation.
- Placeholder text must be `Select {noun}` format — never `Choose`, never `Pick`.

## Hook Contract: useRefData

Not a visual surface, but the executor needs the exact API to implement `<RefSelect>` correctly.

```js
// frontend/src/hooks/useRefData.js
import { useQuery } from '@tanstack/react-query';
import { getRefData } from '@/api/refData';

export function useRefData(category) {
  return useQuery({
    queryKey: ['ref', category],
    queryFn: () => getRefData(category),
    staleTime: 5 * 60 * 1000,  // 5 minutes — per ADMIN-07
    enabled: Boolean(category)
  });
}
```

Query key `['ref', category]` is the canonical pattern for all downstream phases (ADMIN-07, ADMIN-08). The executor must not deviate from this key format.
