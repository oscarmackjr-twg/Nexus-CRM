## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip gap, inline badge padding |
| sm | 8px | Compact element spacing, button inner gap |
| md | 16px | Default element spacing, card padding sides |
| lg | 24px | Section padding, card-to-card gap |
| xl | 32px | Layout grid gap |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Tailwind class mapping (match existing patterns):**

| Tailwind | Value | Existing usage |
|----------|-------|----------------|
| `p-4` | 16px | CardContent padding ‚Äî use unchanged |
| `space-y-3` | 12px | Field list inside a card section |
| `space-y-4` | 16px | Card-to-card within a column |
| `space-y-6` | 24px | Page-level section stacking |
| `gap-6` | 24px | Grid column gap (`xl:grid-cols-[320px,1fr]`) |
| `gap-2` | 8px | Chip row gap, badge row gap |

Exceptions:
- Employment history and board membership rows: `gap-3` (12px) between table columns within a row.
- Amount + currency side-by-side pair: `flex gap-2` ‚Äî amount input `flex-1`, currency input `w-16`.
- Touch targets (icon-only buttons): `h-10 w-10` (40px) ‚Äî matches existing Button `size="icon"`.

Source: codebase scan of `ContactDetailPage.jsx` and `CompanyDetailPage.jsx`.

## Screen-by-Screen Interaction Contract

### ContactDetailPage ‚Äî Profile Tab

**Tab order:** Profile | Timeline | Deals | Tasks | LinkedIn

The Profile tab is added as `defaultValue="profile"` ‚Äî it opens by default when navigating to a contact.

**Card 1 ‚Äî Identity** (`CardTitle`: "Identity")

Sub-sections within the card (separated by `text-xs font-medium text-muted-foreground uppercase tracking-wide` section header):

_Contact details sub-section:_
- `contact_type_id` ‚Äî `<RefSelect category="contact_type" />` single select, label "Contact type"
- `primary_contact` ‚Äî `<Switch />` with label "Primary contact", badge `variant="accent"` shown next to name when true
- `business_phone` ‚Äî `<Input type="tel" />`, label "Business phone"
- `mobile_phone` ‚Äî `<Input type="tel" />`, label "Mobile phone"
- `linkedin_url` ‚Äî `<Input type="url" />`, label "LinkedIn URL"; display mode shows clickable link with `ExternalLink` icon (4px gap)

_Address sub-section:_
- `address` ‚Äî `<Input />`, label "Street address"
- `city` + `state` ‚Äî two `<Input />` side-by-side (`flex gap-2`)
- `postal_code` + `country` ‚Äî two `<Input />` side-by-side (`flex gap-2`)

_Employment History sub-section_ (standalone card or sub-section ‚Äî Claude decision: **standalone Card** titled "Employment history" because 4+ columns per row warrant their own card):
- Row UI: each row = `Company` (Input flex-1) | `Title` (Input flex-1) | `From` (Input w-20, year) | `To` (Input w-20, year, placeholder "present")
- `√ó` button (`Button variant="ghost" size="icon"`) at row end removes the row
- `+ Add position` button (`Button variant="outline" size="sm"`) below the row list appends a new empty row
- Empty state: `text-sm text-muted-foreground` ‚Äî "No previous employment recorded."

_Board Memberships sub-section_ (standalone Card titled "Board memberships"):
- Row UI: each row = `Company` (Input flex-1) | `Title` (Input flex-1)
- Same `√ó` and `+ Add board seat` pattern as employment history
- Empty state: "No board memberships recorded."

**Card 2 ‚Äî Investment Preferences** (`CardTitle`: "Investment Preferences")
- `sector` (multi-ref JSONB) ‚Äî chips + RefSelect pattern: existing chips render as `Badge` with `√ó Button variant="ghost" size="icon"`; `<RefSelect category="sector" />` below chips appends a value. Label "Sectors"
- `sub_sector` (multi-ref JSONB) ‚Äî same chips pattern. Label "Sub-sectors"
- `contact_frequency` ‚Äî `<Input type="number" min="1" />`, label "Contact frequency (days)"

**Card 3 ‚Äî Internal Coverage** (`CardTitle`: "Internal Coverage")
- `coverage_persons` (M2M users) ‚Äî chips pattern sourced from `/users` endpoint (not ref_data). Each chip = display name + `√ó`. Below chips: user search `<Input placeholder="Add coverage person..." />` that filters users list. Label "Coverage persons"
- `legacy_id` ‚Äî `<Input />`, label "Legacy ID"

**Edit mode:** The existing inline edit pattern (`editing` boolean state, pencil icon in CardHeader) is extended per-card. Each card has its own edit toggle (`Button variant="ghost" size="icon"` with `Edit3` icon) in the `CardHeader`. Saving one card does not affect others. Save button: `Button variant="default"` full-width within the card form, label "Save changes".

- `sub_sector_preferences` (multi JSONB) ‚Äî chips + RefSelect for `sub_sector`

## Interaction States

| State | Treatment |
|-------|-----------|
| Loading (Profile tab initial fetch) | `<Skeleton className="h-[400px] w-full" />` ‚Äî single block per card area |
| Saving (mutation pending) | Save button: `disabled + opacity-50` (built into Button component) |
| RefSelect loading | Spinner inside select ‚Äî existing RefSelect loading state (Phase 2) |
| RefSelect error | "Could not load options" inside select ‚Äî existing RefSelect error state |
| RefSelect empty | "No options available" inside select ‚Äî existing RefSelect empty state |
| Chip √ó hover | `hover:bg-muted` on the √ó ghost button ‚Äî existing ghost variant |
| Switch on | Accent color fill ‚Äî CSS variable `--accent` via Tailwind `bg-accent` |
| Employment row inline edit | Row always in edit mode when editing card; fields are enabled/disabled with card's `editing` toggle |
| Amount input invalid (non-numeric) | Browser native `type="number"` validation; no custom error UI required for this phase |
