## Spacing Scale

**Project spacing set for this codebase: 4, 8, 12, 16, 24, 32, 48, 64** — 12px is retained from existing `space-y-3` / `gap-3` patterns established in Phase 1 and present throughout `ContactDetailPage` and `CompanyDetailPage`. 12px is an approved project extension to the standard 8-point scale, not an undeclared non-standard value.

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip gap, inline badge padding |
| sm | 8px | Compact element spacing, button inner gap |
| sm-plus | 12px | Field list stacking within a card, structured row column gaps — **approved project extension** |
| md | 16px | Default element spacing, card padding sides |
| lg | 24px | Section padding, card-to-card gap |
| xl | 32px | Layout grid gap |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Tailwind class mapping (match existing patterns):**

| Tailwind | Value | Existing usage |
|----------|-------|----------------|
| `p-4` | 16px | CardContent padding — use unchanged |
| `space-y-3` | 12px | Field list inside a card section (`sm-plus` token) |
| `space-y-4` | 16px | Card-to-card within a column |
| `space-y-6` | 24px | Page-level section stacking |
| `gap-6` | 24px | Grid column gap (`xl:grid-cols-[320px,1fr]`) |
| `gap-3` | 12px | Structured row column gaps (`sm-plus` token) |
| `gap-2` | 8px | Chip row gap, badge row gap |

Exceptions:
- Employment history and board membership rows: `gap-3` (12px / `sm-plus` token) between table columns within a row.
- Amount + currency side-by-side pair: `flex gap-2` — amount input `flex-1`, currency input `w-16`.
- Touch targets (icon-only buttons): `h-10 w-10` (40px) — matches existing Button `size="icon"`.

Source: codebase scan of `ContactDetailPage.jsx` and `CompanyDetailPage.jsx`.

## Screen-by-Screen Interaction Contract

### ContactDetailPage — Profile Tab

**Primary visual anchor:** The contact name heading (`text-3xl font-semibold`) rendered above the tab bar is the page-level focal point. All other content is subordinate to this element.

**Tab order:** Profile | Timeline | Deals | Tasks | LinkedIn

The Profile tab is added as `defaultValue="profile"` — it opens by default when navigating to a contact.

**Card 1 — Identity** (`CardTitle`: "Identity")

Sub-sections within the card (separated by `text-xs font-semibold text-muted-foreground uppercase tracking-wide` section header):

_Contact details sub-section:_
- `contact_type_id` — `<RefSelect category="contact_type" />` single select, label "Contact type"
- `primary_contact` — `<Switch />` with label "Primary contact", badge `variant="accent"` shown next to name when true
- `business_phone` — `<Input type="tel" />`, label "Business phone"
- `mobile_phone` — `<Input type="tel" />`, label "Mobile phone"
- `linkedin_url` — `<Input type="url" />`, label "LinkedIn URL"; display mode shows clickable link with `ExternalLink` icon (4px gap); icon button `aria-label="Open LinkedIn profile"`

_Address sub-section:_
- `address` — `<Input />`, label "Street address"
- `city` + `state` — two `<Input />` side-by-side (`flex gap-2`)
- `postal_code` + `country` — two `<Input />` side-by-side (`flex gap-2`)

_Employment History sub-section_ (standalone card or sub-section — Claude decision: **standalone Card** titled "Employment history" because 4+ columns per row warrant their own card):
- Row UI: each row = `Company` (Input flex-1) | `Title` (Input flex-1) | `From` (Input w-20, year) | `To` (Input w-20, year, placeholder "present")
- `×` button (`Button variant="ghost" size="icon"`) at row end removes the row; `aria-label="Remove position"`
- `+ Add position` button (`Button variant="outline" size="sm"`) below the row list appends a new empty row
- Empty state: `text-sm text-muted-foreground` — "No previous employment recorded."

_Board Memberships sub-section_ (standalone Card titled "Board memberships"):
- Row UI: each row = `Company` (Input flex-1) | `Title` (Input flex-1)
- Same `×` (`aria-label="Remove board seat"`) and `+ Add board seat` pattern as employment history
- Empty state: "No board memberships recorded."

**Card 2 — Investment Preferences** (`CardTitle`: "Investment Preferences")
- `sector` (multi-ref JSONB) — chips + RefSelect pattern: existing chips render as `Badge` with `× Button variant="ghost" size="icon"` (`aria-label="Remove {value}"`); `<RefSelect category="sector" />` below chips appends a value. Label "Sectors"
- `sub_sector` (multi-ref JSONB) — same chips pattern. Label "Sub-sectors"
- `contact_frequency` — `<Input type="number" min="1" />`, label "Contact frequency (days)"

**Card 3 — Internal Coverage** (`CardTitle`: "Internal Coverage")
- `coverage_persons` (M2M users) — chips pattern sourced from `/users` endpoint (not ref_data). Each chip = display name + `×` (`aria-label="Remove {name}"`). Below chips: user search `<Input placeholder="Add coverage person..." />` that filters users list. Label "Coverage persons"
- `legacy_id` — `<Input />`, label "Legacy ID"

**Edit mode:** The existing inline edit pattern (`editing` boolean state, pencil icon in CardHeader) is extended per-card. Each card has its own edit toggle (`Button variant="ghost" size="icon"` with `Edit3` icon, `aria-label="Edit {card name}"`) in the `CardHeader`. Saving one card does not affect others. Save button: `Button variant="default"` full-width within the card form, label "Save changes".

### CompanyDetailPage — Profile Tab

**Primary visual anchor:** The company name heading (`text-3xl font-semibold`) rendered above the tab bar is the page-level focal point. All other content is subordinate to this element.

**Tab order:** Profile | Contacts | Deals | LinkedIn

**Card 1 — Identity** (`CardTitle`: "Identity")
- `company_type_id` — `<RefSelect category="company_type" />` single select

- `tier_id` — `<RefSelect category="tier" />` single select
- `sector_id` — `<RefSelect category="sector" />` single select
- `sub_sector_id` — `<RefSelect category="sub_sector" />` single select
- `description` — `<Textarea rows={3} />`, label "Description"
- `main_phone` — `<Input type="tel" />`, label "Main phone"
- `parent_company_id` — company search `<Input placeholder="Search parent company..." />` with typeahead (queries `/companies`); displays selected company name as chip with `×` (`aria-label="Remove parent company"`)
- `address` — `<Input />`, label "Street address"
- `city` + `state` — two `<Input />` side-by-side
- `postal_code` + `country` — two `<Input />` side-by-side

**Card 2 — Investment Profile** (`CardTitle`: "Investment Profile")

_AUM & Financials sub-section:_
- `aum_amount` + `aum_currency` — amount `<Input type="number" className="flex-1" />` + currency `<Input className="w-16" maxLength={3} />` side-by-side, label "AUM"
- `ebitda_amount` + `ebitda_currency` — same pattern, label "EBITDA"
- `min_ebitda` + `max_ebitda` + `ebitda_range_currency` — three inputs: `<Input type="number" className="flex-1" />` (min) + `<Input type="number" className="flex-1" />` (max) + `<Input className="w-16" />` (currency), label "EBITDA range"

_Deal preferences sub-section:_
- `typical_bite_size_low` + `typical_bite_size_high` + `bite_size_currency` — three inputs as EBITDA range, label "Typical bite size"
- `co_invest` — `<Switch />`, label "Co-invest"
- `transaction_types` (multi JSONB) — chips (`aria-label="Remove {value}"`) + RefSelect for `transaction_type` category
- `sector_preferences` (multi JSONB) — chips (`aria-label="Remove {value}"`) + RefSelect for `sector`
- `sub_sector_preferences` (multi JSONB) — chips (`aria-label="Remove {value}"`) + RefSelect for `sub_sector`
- `preference_notes` — `<Textarea rows={3} />`, label "Preference notes"

**Card 3 — Internal** (`CardTitle`: "Internal")
- `watchlist` — `<Switch />`, label "Watchlist"; badge `variant="accent"` on company name when true
- `coverage_person_id` — single user select `<Select />` populated from `/users`, label "Coverage person"
- `contact_frequency` — `<Input type="number" min="1" />`, label "Contact frequency (days)"
- `legacy_id` — `<Input />`, label "Legacy ID"

## Interaction States

| State | Treatment |
|-------|-----------|
| Loading (Profile tab initial fetch) | `<Skeleton className="h-[400px] w-full" />` — single block per card area |
| Saving (mutation pending) | Save button: `disabled + opacity-50` (built into Button component) |
| RefSelect loading | Spinner inside select — existing RefSelect loading state (Phase 2) |
| RefSelect error | "Could not load options" inside select — existing RefSelect error state |
| RefSelect empty | "No options available" inside select — existing RefSelect empty state |
| Chip × hover | `hover:bg-muted` on the × ghost button — existing ghost variant |
| Switch on | Accent color fill — CSS variable `--accent` via Tailwind `bg-accent` |
| Employment row inline edit | Row always in edit mode when editing card; fields are enabled/disabled with card's `editing` toggle |
| Amount input invalid (non-numeric) | Browser native `type="number"` validation; no custom error UI required for this phase |
